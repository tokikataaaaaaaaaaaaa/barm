# Feature Brainstorm: Time-Boxed Challenge

**Feature Name:** `time-boxed-challenge`
**Service:** `barm`
**Phase:** 1 - Brainstorming
**Created:** 2026-01-03
**Status:** Complete

---

## 1. 課題の明確化

### 現在の課題

| 課題 | 詳細 |
|------|------|
| **差別化不足** | 現在の「個人ミッション + グループ」モデルは Todo アプリと大差ない |
| **仲間意識の欠如** | グループ機能があっても「一緒に頑張ってる」感が薄い |
| **コミットメント不足** | 「いつでも始められる＝いつでもやめられる」心理 |
| **課金タイミングの壁** | 無料→有料の転換でユーザーが離脱しやすい |

### ターゲットユーザー

- **Primary:** 20〜30代社会人男性（筋トレ・勉強を続けたいが一人だと続かない）
- **Secondary:** 10代学生男性（部活・受験勉強の両立）

### 解決した場合のインパクト

| 指標 | Before | After (期待) |
|------|--------|--------------|
| 7日継続率 | 40% | 70%+ |
| 課金転換率 | 5% | 15%+ |
| NPS | +20 | +40+ |

## 2. アイデア発散

### アイデア A: Time-Boxed Challenge (推奨)

**概要:** 期間限定チャレンジ（1週間/2週間/1ヶ月）に参加し、同期で始めた仲間と一緒にゴールを目指す

**仕組み:**
1. トップページにチャレンジ一覧を表示
2. 「参加する」で期間中の目標を設定
3. 同時期に参加した仲間のプロフィール・目標が見える
4. チャレンジ終了後、次のチャレンジ参加時に課金

**Pros:**
- ✅ 「コホート感」が強い（同時スタート）
- ✅ 期限があるからコミットしやすい
- ✅ 成功体験の後に課金 = 転換率UP
- ✅ Todo アプリと明確に差別化

**Cons:**
- ⚠️ 参加者が少ないとチャレンジが成立しない
- ⚠️ チャレンジ開始タイミングの設計が必要

---

### アイデア B: Weekly Reset Challenge

**概要:** 毎週月曜日に全員がリセットされ、その週の目標を立てる

**仕組み:**
1. 毎週月曜 0:00 に全員の記録がリセット
2. その週の目標を各自設定
3. 日曜日に結果発表・ランキング

**Pros:**
- ✅ シンプルで理解しやすい
- ✅ 毎週「再スタート」できる
- ✅ 脱落しても復帰しやすい

**Cons:**
- ⚠️ 1週間の縛りが短すぎる可能性
- ⚠️ 毎週の繰り返しで飽きる可能性
- ⚠️ 「仲間と一緒に」感が薄い

---

### アイデア C: Ladder Challenge

**概要:** 1週間 → 2週間 → 1ヶ月と段階的にレベルアップしていく

**仕組み:**
1. 1週間チャレンジ成功 → 2週間に昇格
2. 2週間成功 → 1ヶ月に昇格
3. 上位チャレンジほど参加者が絞られる

**Pros:**
- ✅ ゲーミフィケーション要素
- ✅ 上位者は「精鋭」感が出る
- ✅ 段階的に難易度が上がる

**Cons:**
- ⚠️ 失敗時の再スタートが面倒
- ⚠️ 上位に行くほど参加者が減る
- ⚠️ 複雑さが増す

## 4. 実現可能性評価

### 技術的複雑さ

| 項目 | 複雑さ | 備考 |
|------|--------|------|
| チャレンジ CRUD | 低 | 既存の Mission モデルを拡張 |
| 参加者マッチング | 中 | Firestore クエリで実現可能 |
| 課金フロー変更 | 中 | RevenueCat の既存実装を活用 |
| リアルタイム表示 | 低 | 既存のライブフィードを活用 |

### 必要なリソース

| 作業 | 工数見積もり |
|------|-------------|
| データモデル設計 | 1日 |
| UI デザイン | 2日 |
| フロントエンド実装 | 3日 |
| バックエンド（Firestore ルール） | 1日 |
| テスト | 1日 |
| **合計** | **約8日** |

### リスク

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 参加者不足でチャレンジが寂しい | 高 | 最低3人で開始 or ダミー表示 |
| 課金タイミングでの離脱 | 中 | 「次も一緒に」の訴求強化 |
| 既存機能との整合性 | 低 | 段階的移行 |

## 5. 推奨アイデア

### 選定: アイデア A - Time-Boxed Challenge

**選定理由:**

1. **ユーザー課題に直結**
   - 「一人だと続かない」→「同時スタートの仲間」
   - 「いつでもやめられる」→「期限付きコミット」

2. **ビジネス的に有利**
   - 成功後課金 = 課金転換率UP
   - 価格設定が柔軟（100〜300円/チャレンジ）

3. **競合との差別化が明確**
   - vs Cohorty: 日本語 + 男性特化 + 成功後課金
   - vs みんチャレ: チャットなし + シンプル

4. **技術的に実現可能**
   - 既存のモデル・UIを拡張
   - 8日程度で実装可能
