## 1. ユーザーフロー

### 1.1 メインフロー（Happy Path）

```
[ホーム画面]
     │
     ▼
[チャレンジ一覧 C-001]  ←── 「チャレンジ」タブをタップ
     │
     │ チャレンジカードをタップ
     ▼
[チャレンジ参加 C-002]  ←── 目標設定モーダル
     │
     │ 「参加する」をタップ
     ▼
[チャレンジ詳細 C-003]  ←── 参加者一覧・記録入力
     │
     │ 毎日記録を入力
     │ （期間終了まで繰り返し）
     ▼
[チャレンジ結果 C-004]  ←── 完走判定・結果表示
     │
     │ 「次に挑戦する」をタップ
     ▼
[課金確認 C-005]  ←── 次回チャレンジ選択・決済
     │
     │ 支払い完了
     ▼
[チャレンジ詳細 C-003]  ←── 新しいチャレンジ開始
```

### 1.2 サブフロー

#### 初回無料フロー
```
[チャレンジ参加 C-002]
     │
     │ isPaid: false (初回)
     │ → 料金表示なし
     ▼
[チャレンジ詳細 C-003]
```

#### リタイア/途中離脱フロー
```
[チャレンジ詳細 C-003]
     │
     │ 期間終了（達成率 < 80%）
     ▼
[チャレンジ結果 C-004]
     │
     │ 「惜しい！」または「リタイア」表示
     │ 次回チャレンジへの誘導
     ▼
[課金確認 C-005]
```

## 2. 画面状態定義

### 2.1 状態マトリクス

| 画面 | Loading | Empty | Loaded | Error |
|------|---------|-------|--------|-------|
| C-001 チャレンジ一覧 | スケルトン3枚 | 「チャレンジ準備中」 | チャレンジカード表示 | リトライボタン |
| C-002 チャレンジ参加 | - | - | 目標入力フォーム | エラートースト |
| C-003 チャレンジ詳細 | スケルトン | - | 参加者・記録表示 | リトライボタン |
| C-004 チャレンジ結果 | - | - | 結果・お祝い表示 | - |
| C-005 課金確認 | 決済処理中 | - | 料金・プラン表示 | 決済エラー |

### 2.2 状態遷移

```
[Initial] → [Loading] → [Loaded]
                    ↘ [Error] → [Retry] → [Loading]

[Loaded] → [Empty] (データなし時)
```

## 5. インタラクション定義

### 5.1 アニメーション

| 要素 | トリガー | アニメーション |
|------|----------|----------------|
| チャレンジカード | ホバー | `scale(1.02)`, `translateY(-4px)` |
| 参加ボタン | タップ | `scale(0.95)` + ハプティック |
| 参加成功 | 成功 | 紙吹雪エフェクト (confetti) |
| 目標達成 | 記録保存 | ゴールドグロウ + バイブレーション |
| 完走 | 結果表示 | お祝いアニメーション (Lottie) |
| 参加者カード | 達成時 | チェックマーク bounce in |

### 5.2 マイクロインタラクション

```
記録入力:
  [-] ボタン: タップでデクリメント、長押しで連続
  [+] ボタン: タップでインクリメント、長押しで連続

  目標達成時:
    - 数値がゴールド色に変化
    - チェックマークが bounce in
    - 軽いバイブレーション
    - 「達成！」トースト表示

参加者一覧:
  - スクロール時、自分の行は sticky (常に見える)
  - 達成者は背景が緑がかる
  - 未達成者はそのまま

完走判定:
  - 80%以上: 🎉 お祝いアニメーション + confetti
  - 50-79%: 💪 「惜しい！」表示
  - 50%未満: 励ましメッセージ
```

### 5.3 トランジション

| From | To | トランジション |
|------|-----|----------------|
| チャレンジ一覧 | 参加モーダル | `fadeIn` + `slideUp` |
| 参加モーダル | チャレンジ詳細 | `fadeIn` (モーダル閉じ) → ページ遷移 |
| チャレンジ詳細 | 結果画面 | `fadeIn` + お祝いアニメ |
| 結果画面 | 課金確認 | `slideIn` (右から) |

### 5.4 エラーハンドリング

| エラー | 表示 | アクション |
|--------|------|------------|
| 参加失敗 | トースト「参加できませんでした」 | リトライボタン |
| 記録保存失敗 | トースト「保存に失敗しました」 | 自動リトライ (3回) |
| 決済失敗 | モーダル「決済が完了しませんでした」 | 再試行 or キャンセル |
| ネットワークエラー | オフラインバナー | 接続復帰後に自動同期 |
